Design patterns syllabus

Learn by category, revise by signal.

Creational

Creation patterns help you decide where construction happens, how many instances exist, and how product families stay consistent.

SingletonKeeps one shared instance and one controlled access point for code that truly needs to stay singular.BuilderSeparates the steps of construction from the final object so you can create complex products without a giant constructor.FactoryMoves the choice of concrete type away from callers so object creation follows intention instead of `new` scattered everywhere.Abstract FactoryCreates families of related objects that are designed to work together and share the same theme.PrototypeBuilds new objects by copying an existing one and then tailoring the clone.

Structural

Structural patterns help objects fit together cleanly, whether that means translating interfaces, stacking behavior, or sharing memory.

AdapterTranslates one interface into another so existing code can work with a mismatched dependency.CompositeTreats individual objects and groups of objects the same way by placing them in a tree-shaped hierarchy.DecoratorAdds responsibilities by wrapping an object in layers instead of subclassing it into many combinations.ProxyStands in front of a real object and controls how, when, or whether the object is accessed.FlyweightShares common intrinsic state so a huge number of tiny objects can stay memory efficient.

Behavioral

Behavioral patterns help code decide who reacts, who handles, and how algorithms or states evolve over time.

Chain of ResponsibilityLets a request travel through a sequence of handlers until one decides to act on it.StateLets an object alter its behavior when its internal state changes, without giant conditional blocks.ObserverBroadcasts changes from one subject to many listeners that care about updates.StrategyEncapsulates interchangeable algorithms so the caller can swap the behavior without changing the surrounding code.Template MethodDefines the skeleton of an algorithm while letting subclasses fill in selected steps.

Depth checklist

  • Can I identify the pain that created the pattern?
  • Can I explain the trade-off without using jargon?
  • Can I compare it with a neighboring pattern?
  • Can I name one practical interview example?
